# Contributing to Sid Trading Lab

Thank you for your interest in contributing! This is a demo-first trading research lab — all contributions help improve safety, education, and tooling.

## 🚀 Quick Start

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-name`
3. Make changes with tests
4. Run `pytest` to verify
5. Submit a pull request

## 📋 Guidelines

### Code Style
- Python 3.14+ type hints required
- `ruff` for linting, `black` for formatting
- Docstrings for all public functions

### Safety First
- Never submit real API credentials
- All new strategies must include backtests
- Maintain demo-only defaults

### What We Need
- 📜 Documentation fixes
- 🐛 Bug reports with reproduction steps
- 🎲 New strategies (with Sharpe/CAGR backtest results)
- 🎨 Dashboard / UI improvements
- 🧪 Test coverage improvements

## 🏷️ Issue Labels

| Label | Use |
|---|---|
| `bug` | Something is broken |
| `enhancement` | New feature or improvement |
| `documentation` | Docs need work |
| `strategy` | New or improved trading strategy |
| `good first issue` | Easy entry point for new contributors |
